#Resumo

  O desenvolvimento de jogos é um processo multidisciplinar que envolve planejamento, programação, design, testes e publicação. Você pode iniciar seus estudos seguindo esses passos

1. **Introdução ao Desenvolvimento de Jogos** - Etapas de desenvolvimento e as diferentes áreas envolvidas (programação, arte, design, áudio e testes).

2. **Planejamento e Game Design** - Gênero, público-alvo, mecânicas, narrativa, personagens, ambientação, interface e documentação do jogo.

3. **Lógica de Programação e Programação Orientada a Objetos (POO)** - Algoritmos, variáveis, estruturas condicionais, laços de repetição, funções e os conceitos fundamentais de POO.

4. **Escolha da Game Engine** - Conheça as principais engines (Godot, Unity, Unreal Engine e GameMaker), suas vantagens, limitações e para quais tipos de projetos cada uma é mais indicada.

5. **Física e Interação** - Colisões, gravidade, movimentação, animações, eventos e entrada de comandos do jogador.

6. **Desenvolvimento de Projetos** - Coloque o conhecimento em prática criando pequenos jogos, isso pode te ajudar a aprender lidar com diversos desafios durante a programação do jogo.

## Glossário

| Termo | Definição |
|--------|-----------|
| **Game Engine** | Plataforma de software que reúne ferramentas como renderização, física, áudio e animação para facilitar o desenvolvimento de jogos. |
| **GDD (Game Design Document)** | Documento que descreve o projeto do jogo, incluindo mecânicas, narrativa, personagens, regras, objetivos e demais informações necessárias para o desenvolvimento. |
| **POO (Programação Orientada a Objetos)** | Paradigma de programação que organiza o código em classes e objetos, promovendo reutilização, organização e facilidade de manutenção. |
| **Sprite** | Imagem ou animação bidimensional utilizada para representar personagens, objetos ou elementos visuais em jogos 2D. |
| **Collider (Colisor)** | Componente responsável por definir a área de colisão de um objeto, permitindo detectar interações físicas entre elementos do jogo. |
| **Script** | Arquivo contendo o código responsável por controlar o comportamento de personagens, objetos e sistemas dentro do jogo. |
| **Rigidbody** | Componente que aplica propriedades físicas a um objeto, permitindo que ele seja afetado por gravidade, forças e colisões. |
| **Protótipo** | Primeira versão funcional de um jogo, desenvolvida para validar ideias, mecânicas e a experiência do jogador antes da produção completa. |
| **Storyboard** | Representação visual das principais cenas e eventos do jogo, utilizada para planejar a narrativa e o fluxo da experiência do jogador. |
| **Assets** | Recursos utilizados no desenvolvimento do jogo, como imagens, sprites, modelos 3D, animações, sons, músicas, efeitos visuais e fontes. |
